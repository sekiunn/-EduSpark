\# 功能1：企业级页面从零创建全流程（商用标准，可直接复用）

\## 输入条件（明确输入，避免AI无依据生成）

项目名称、页面类型（官网/中后台/SaaS等）、品牌VI规范、目标用户、核心功能、合规要求（如无障碍、SEO）

\## 执行步骤（严格顺序，不可颠倒）+ 标准输出物模板（每步必出，可交付）

\### Step1：商业定调（核心第一步，定灵魂，根治AI味）

\- 执行动作：明确品牌定位、目标人群、核心情绪词、视觉风格、禁用元素，对齐品牌VI

\- 输出物：《页面调性文档》（可直接交付）

\- 模板：

&#x20; 1.  项目定位：XXX（如“企业SaaS数据管理平台”）

&#x20; 2.  目标人群：XXX（如“企业运维人员、数据分析师”）

&#x20; 3.  核心情绪词：XXX（3个以内，如“专业、高效、极简”）

&#x20; 4.  视觉风格：XXX（如“磨砂玻璃+极简金属，无过度装饰”）

&#x20; 5.  禁用元素：XXX（如“高饱和色彩、浮夸动效、纯平塑料感组件”）



\### Step2：规范生成（统一设计标准，避免混乱）

\- 执行动作：基于调性文档，生成量化设计Token，统一色彩、字体、间距、材质规范

\- 输出物：《设计Token清单》（可直接复用，团队共用）

\- 模板：

&#x20; 1.  色值表：主色（#XXX）、辅助色1（#XXX）、辅助色2（#XXX）、中性色1（#XXX）、中性色2（#XXX）、中性色3（#XXX）

&#x20; 2.  字体表：字号（12/14/16/20/24/32px）、字体权重、行高

&#x20; 3.  间距表：模块间距、内边距、外边距（均为8的倍数）

&#x20; 4.  材质光影参数：纹理透明度、阴影模糊度、阴影不透明度



\### Step3：布局设计（响应式优先，企业级排版）

\- 执行动作：基于设计Token，设计三端（PC/平板/手机）响应式布局，梳理信息层级，划分功能模块

\- 输出物：《响应式布局草图》（可直接用于开发）

\- 模板：

&#x20; 1.  PC端（>1200px）：布局结构（如“顶部导航+左侧侧边栏+中间内容区+右侧操作区”）、模块划分、信息层级

&#x20; 2.  平板端（768px-1200px）：布局适配调整（如“隐藏侧边栏，折叠导航”）、模块适配规则

&#x20; 3.  移动端（<768px）：布局简化（如“顶部导航折叠+内容区单列展示”）、触控区域调整



\### Step4：组件落地（复用商用组件，统一质感）

\- 执行动作：复用component-css.md中的商用组件，映射功能模块与组件，统一交互反馈

\- 输出物：《页面组件映射表》（开发直接对照）

\- 模板：

&#x20; 1.  功能模块：XXX（如“数据统计卡片”）

&#x20; 2.  对应商用组件：XXX（如“card-commercial”）

&#x20; 3.  交互规则：XXX（如“点击卡片进入详情页，hover无动效，点击有按压反馈”）

&#x20; 4.  材质适配：XXX（如“添加磨砂纹理，柔光阴影”）



\### Step5：动效优化（克制微动效，无浮夸）

\- 执行动作：仅添加必要的交互反馈动效，删除冗余动画，统一动效参数

\- 输出物：《微动效清单》（开发直接复用）

\- 模板：

&#x20; 1.  触发场景：XXX（如“按钮按压、页面加载、表单提交”）

&#x20; 2.  动画类型：XXX（如“轻微下沉、淡入淡出、加载旋转”）

&#x20; 3.  时长：XXX（如“0.2s，统一时长，无随机动画”）

&#x20; 4.  禁用项：XXX（如“无转场动画、无自动轮播、无外发光动效”）



\### Step6：工程输出（企业级代码，可直接上线）

\- 执行动作：基于布局草图、组件映射表，生成语义化HTML、模块化CSS、响应式适配代码，适配SEO、无障碍

\- 输出物：《标准工程代码》（可直接运行、交付）

\- 模板：

&#x20; 1.  语义化HTML：使用header/nav/main/section/footer等标签，无div滥用

&#x20; 2.  模块化CSS：复用component-css.md中的组件样式，统一设计Token引用

&#x20; 3.  响应式适配代码：媒体查询（对应三端断点）、流式布局、触控区域适配

&#x20; 4.  额外适配：图片alt文本、链接title、键盘操作支持



\### Step7：合规校验（企业级验收，确保可上线）

\- 执行动作：使用推荐工具检测SEO、无障碍、性能，人工校准细节，消除AI残留

\- 输出物：《合规校验报告》（可作为上线依据）

\- 模板：

&#x20; 1.  SEO评分：XXX（如“Lighthouse SEO评分90+”）、问题及修改建议

&#x20; 2.  无障碍检测：XXX（如“axe DevTools检测无严重问题”）、问题及修复方案

&#x20; 3.  性能指标：XXX（如“页面加载时间<2s，LCP<1.5s”）、优化建议

&#x20; 4.  视觉校准：XXX（如“无AI塑料感、无模板组件、质感统一”）
